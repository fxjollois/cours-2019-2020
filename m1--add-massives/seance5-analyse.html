ACP¶

Dans le sous-module decomposition, nous allons importer la fonction PCA(). Le fonctionnement de celle-ci est similaire toutes les autres dans scikit-learn.

  1. On cr d'abord un objet (nomm ici pca) qui va contenir le rsultat de l'ACP. Dans la fonction PCA(), on pourra indiquer les paramtres tels que le nombre de composants retenir (n_components) ou la mthode de calcul (svd_solver).
  2. Ensuite, on ajuste (fit en anglais) sur des donnes avec la fonction fit() de l'objet prcdemment cr. Dans cette fonction, nous devons donc passer les donnes utiliser.

Si l'on souhaite une ACP norme, il nous faut standardiser les donnes en amont, avec la fonction scale() du sous-module preprocessing (importe aussi ici). Bien videmment, il ne faut passer que des variables quantitatives.

In [3]:
from sklearn.decomposition import PCA
from sklearn.preprocessing import scale

pca = PCA()
pca.fit(scale(iris[iris.columns[:4]]))

DBSCAN¶

DBSCAN est un algorithme de classification non supervise, bas sur la densit. Il est intressant car il ne ncessite pas de connatre le nombre de classes, mais une estimation de la densit (globale) des donnes. En effet, les points proches (distance infrieure $\varepsilon$) sont considrs dans la mme classe. Si toutefois cette classe comporte au moins un cetain nombre de points au final. Si ce n'est pas le cas, les points sont considrs comme outliers et mis part.

Nous utilisons ici la fonction DBSCAN() du sous-module cluster. Celle-ci peut prendre en paramtre, entre autres, la valeur de $\varepsilon$ (eps, .5 par dfaut) et le nombre minimal d'objets dans une classe (min_samples, 5 par dfaut).

Puis nous ajustons les donnes standardises (pour ne pas donner plus d'influences une variable qu' une autre).

In [11]:
from sklearn.cluster import DBSCAN

db = DBSCAN(eps = .6, min_samples = 3)
db.fit(scale(iris[iris.columns[:4]]))
Out[11]:
DBSCAN(algorithm='auto', eps=0.6, leaf_size=30, metric='euclidean',
    metric_params=None, min_samples=3, n_jobs=1, p=None)
